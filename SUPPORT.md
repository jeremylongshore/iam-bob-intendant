# Support

## Getting Help

- **Documentation**: Check the [README](README.md) and [000-docs/](000-docs/) directory
- **Bug Reports**: [Open an issue](https://github.com/jeremylongshore/iam-bob-intendant/issues/new?template=bug_report.md)
- **Feature Requests**: [Open an issue](https://github.com/jeremylongshore/iam-bob-intendant/issues/new?template=feature_request.md)
- **Discussions**: [GitHub Discussions](https://github.com/jeremylongshore/iam-bob-intendant/discussions)
- **Security Issues**: See [SECURITY.md](SECURITY.md) — do NOT open public issues

## Support Level

**Bob the Intendant is PRIVATE v0 — there is no support SLA.** Everything below is
best-effort by a solo maintainer, offered as a courtesy, not a commitment. Times
may slip, and there is no promise of a response or a fix on any timeline. Security
reports are the one thing we prioritize — see [SECURITY.md](SECURITY.md).

| Channel | Best-effort target (not a commitment) |
|---------|---------------------------------------|
| Security reports | 24 hours to acknowledge |
| Bug reports | When time allows |
| Feature requests | When time allows |
| General questions | When time allows |

## Before Opening an Issue

1. Search [existing issues](https://github.com/jeremylongshore/iam-bob-intendant/issues) first
2. Check the [documentation](000-docs/)
3. Include version info and reproduction steps (run `bob doctor` and paste the output)
